# v0.7.0 Scope Recommendation

Recommended scope for v0.7.0, based on findings from v0.6.0 (Phases G, A, B, C,
D, E, M, DOC, UX, F). This is a recommendation, not a commitment.

Run the investigation gate equivalent at v0.7.0 start. Verify these priorities
against actual community feedback and KNOWN_UNKNOWNS.md status before locking
scope. The gate's output takes precedence over this document.

Generated: 2026-05-23 (v0.6.0 Phase F Part 2)

---

## High Priority

### 1. Scalar API Explorer (Phase 5b)

The demo is deployed at demo.aevum.build but is not interactive. Scalar API
Explorer would replace the static FastAPI docs with an interactive UI where
developers can send real requests to the governed API from the browser without
a local install. This is the largest remaining DX gap.

Driver: DX (getting-started friction reduction)
Dependency: demo.aevum.build must be stable (confirmed in Phase UX)
Estimated scope: M

### 2. OpenAI Agents and MCP integration guides

The Anthropic and LangChain guides shipped in Phase DOC. The OpenAI Agents
adapter is in CI but has no user-facing guide. The MCP guide covers traceparent
injection but not the full governance workflow for MCP tool calls. Both were
deferred from DOC due to time constraints.

Driver: DX (adopter guidance), Competitive (OpenAI Agents ecosystem)
Estimated scope: S per guide

### 3. VaultTransitSigner live Vault test

The VaultTransitSigner implementation shipped in v0.6.0 but was tested only
with mocks, not a live Vault instance. Before claiming production readiness
for HSM-backed deployments, the signer must be validated against a real
Vault configuration.

Driver: Correctness (documented as untested in v0.6.0 gate reports)
Estimated scope: S (test setup + validation; no code changes expected)

### 4. Grafana Tempo + Langfuse AevumOTelBridge live testing

AevumOTelBridge ships with setup notes for Grafana Tempo and Langfuse, but
both are noted in the docs as untested against live instances. This creates
a support liability if adopters follow the guide and it does not work.

Driver: Correctness (documented as untested), DX
Estimated scope: S (validation pass; likely small doc updates)

### 5. Anthropic SDK adapter: re-evaluate against Stainless-unified SDK

The Stainless-unified Anthropic SDK surface may change in H2 2026. If it
ships, AevumAnthropicAdapter may need a rebuild. Re-evaluate at v0.7.0 start
with an investigation gate: has the unification shipped? If yes, what changed?

Driver: Correctness, Compatibility
Condition: Only act if Stainless unification has shipped by v0.7.0 start date
Estimated scope: S–M depending on API surface change

### 6. Trademark search completion

The TESS (USPTO) and EUIPO trademark searches are required before v1.0 and
have not been initiated. This is a manual, legal-track item. Begin the search
early enough to receive results before v1.0 is planned — trademark searches
typically take weeks, not days.

Driver: Legal (unblock v1.0)
Action: Manual — not a code task
Estimated scope: N/A

## Medium Priority

### 7. OpenClaw adapter Stage 1

The OpenClaw (OpenAI) adapter was deferred from v0.6.0 due to uncertainty
about the hook API stability during and after the OpenAI acquisition. Re-evaluate
at v0.7.0 start: has the hook API stabilized?

Driver: Competitive (OpenAI Agents ecosystem coverage)
Condition: Only begin if hook API stability is confirmed
Estimated scope: M

### 8. Community contribution infrastructure

v0.6.0 ships CONTRIBUTING.md but has not run an external-PR review process.
If v0.6.0 generates external contributions, the project needs a first-PR review
workflow and response SLA documented. Monitor first; act only if needed.

Driver: Community (unknown if v0.6.0 generates external PRs)
Action: Monitor GitHub Issues + PRs after v0.6.0 ships; run working session
        only if external contributions arrive
Estimated scope: S (process docs; no code)

### 9. E-07-PUB: Submit ADR-008 as practitioner abstract

The cross_chain_ref design (ADR-008 — W3C Trace Context + cryptographic causal
linking) is architecturally mature and novel. No draft has been written. The
design will have stronger production validation once v0.7.0 ships multi-agent
A2A integration.

Driver: Thought leadership, community adoption signal
Target venue: practitioner blog or USENIX/IEEE practitioner track
Estimated scope: S (drafting — not a code task)

### 10. ML-DSA hybrid signing implementation

Deferred from v0.6.0 Phase C. The post-quantum signing path (ML-DSA-65) is in
the design but not implemented. FIPS 140-3 coverage for the post-quantum
migration is blocked on this item.

Driver: Cryptography (forward-looking), Compliance (FIPS deferred)
Estimated scope: L (multi-session; requires liboqs-python integration and
                    new conformance tests)

## Deferred (v1.0 or Later)

- **Phase 5c — replay sandbox:** Requires stable policy engine and ledger API
  before a sandboxed replay environment is meaningful.
- **Phase 5d — compliance export tab:** UI complexity; no adopter demand signal.
- **OpenSSF Best Practices Badge (silver tier):** No code changes; significant
  process documentation. Not blocking any adopter use case for v0.7.0.
- **FIPS 140-3 full deployment guide:** Blocked on first regulated customer
  requiring FIPS. Do not write the guide without a real deployment to validate
  against.

## Investigation Gate Checklist for v0.7.0 Start

Run this before locking scope. Gate findings take precedence over this document.

1. Has the Stainless-unified Anthropic SDK shipped? If yes, what changed in the
   message-creation surface that AevumAnthropicAdapter wraps?
2. Has the OpenAI acquisition hook API stabilized? Check OpenAI changelog and
   the Agents SDK release history.
3. Did v0.6.0 generate external GitHub contributions? (Check Issues + PRs.)
4. Is the OG image (currently a placeholder) still unreferenced?
5. What is the current conformance suite test count after any v0.6.x patches?
   Is 74 a sufficient regression baseline for v0.7.0 work?
6. Is demo.aevum.build running cleanly? Check Fly.io dashboard and
   GET /health before starting any DX work.
7. Has E-07-PUB (ADR-008 publication) moved? Check whether any external
   draft or abstract has been started.

Do not skip the gate. The Phase G gate for v0.6.0 prevented all design
reversals across 11 phases (L-06 in LESSONS_LEARNED.md).
